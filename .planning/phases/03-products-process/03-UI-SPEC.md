## Spacing Scale

Declared values (multiples of 4 only). Source: established codebase convention.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators, inline decoration |
| sm | 8px | Label-to-input gap (mb-1.5 = 6px approx; use 8px), tight stacks |
| md | 16px | Default element spacing, form field gaps (space-y-4) |
| lg | 24px | Card internal padding (p-6), section header margin-bottom |
| xl | 32px | Major component gaps |
| 2xl | 48px | Section padding vertical (py-12 minimum) |
| 3xl | 112px | Full section padding (py-28 = established homepage pattern) |

Exceptions:
- Process sticky panel: `top: 0` (viewport-relative, not spacing scale)
- Touch targets: minimum 44px height for all interactive controls (buttons, form inputs)
- Table cells: `py-3 px-4` (12px / 16px) — narrower than standard md to fit 7 columns

## Interaction Contracts

### Form Behavior (all forms)

- Required field validation: HTML5 `required` attribute + `noValidate` on `<form>` element. No custom error styling in Phase 3 (plain browser validation).
- Focus state: `:focus-visible` green-500 outline already in globals.css — do not override.
- Input hover: `border-ink-500/60` (subtle, 150ms transition)
- Input focus: `border-green-500/60` (150ms transition) — matches FooterCTA pattern
- Submit disabled state: not required in Phase 3 (stubbed forms, no async)
- Form submission: `e.preventDefault()` + `console.log(form)` + `setSubmitted(true)`

### TDS Gate Expansion

- Trigger: click "Download TDS →" button
- Transition: Framer Motion `AnimatePresence` — `initial={{ height: 0, opacity: 0 }}` → `animate={{ height: "auto", opacity: 1 }}` over 300ms ease-out
- Download trigger: programmatic anchor element (`document.createElement("a")` → `link.href = "/santosh-tds-placeholder.pdf"` → `link.download = "Santosh-Group-II-Plus-TDS.pdf"` → `link.click()`) fires on form submit before `setSubmitted(true)`
- No scroll-to-form behavior needed (form is already visible at point of click)

### Process Page Scroll Interaction

- `activeStep` state: updated via IntersectionObserver on each right-column step section (threshold 0.5 — fires when step is 50% visible)
- Left panel content: swaps on `activeStep` change with 150ms fade out → 250ms fade in
- Right column step entrance: Framer Motion `useScroll` per step ref, `offset: ["start 0.9", "start 0.4"]`
- Reduced motion: all Framer Motion animations respect `prefers-reduced-motion` via Framer's built-in `useReducedMotion()` — wrap all `motion.div` with `useReducedMotion()` check: if true, pass `style={{ opacity: 1, y: 0 }}` directly
- Progress bar in left panel: CSS transition, not Framer Motion — `transition: width 500ms ease`

### Grade Card CTA Links

- Each grade card "Request Sample →" passes grade pre-selection via query param: `/sample-request?grade=SN150`, `/sample-request?grade=SN500`, `/sample-request?grade=BrightStock`
- The sample request page reads `searchParams` (note: in Next.js 16, `searchParams` is a Promise in async server components — but `SampleRequestForm` is a client component; read via `useSearchParams()` from `next/navigation`)
- Pre-populate the grade select field from `useSearchParams().get("grade")`
