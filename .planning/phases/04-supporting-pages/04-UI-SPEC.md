## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `p-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`, `mb-2`) |
| md | 16px | Default element spacing (`gap-4`, `px-4`, `py-4`, `mb-4`) |
| lg | 24px | Section internal spacing (`gap-6`, `px-6`, `py-6`) |
| xl | 32px | Layout gaps, major element separation (`gap-8`, `py-8`) |
| 2xl | 48px | Section vertical padding (`py-12`) |
| 3xl | 64px | Page-level section breaks (`py-16`, `py-24`, `py-28`) |

Exceptions:
- Section top padding on pages with fixed Nav: `pt-36 lg:pt-40` (first section hero) — consistent with Phase 3 pages
- Form field vertical rhythm: `space-y-5` between fields — matches ContactForm.tsx
- Progress bar height: 12px (`h-3`) — matches SustainabilitySnapshot.tsx

Source: Extracted from globals.css patterns and existing component audit (SustainabilitySnapshot.tsx, ContactForm.tsx).

## Typography

| Role | Font Family | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Body | Barlow | 16px (1rem) | 300 (light) / 400 (regular) | 1.6 | All paragraph text, form inputs, card body copy |
| Label | JetBrains Mono | 12px (0.75rem) | 400 | 1.4 | Section labels (`.section-label`), form field labels (uppercase, `tracking-widest`) |
| Subheading | Barlow Condensed | 16px–20px (1rem–1.25rem) | 600–700 | 1.3 | Card titles, stat labels, button text |
| Heading | Bebas Neue | `clamp(1.75rem, 3vw, 2.5rem)` | 400 (Bebas Neue has single weight) | 1.2 | Section h2 headings |
| Display | Bebas Neue | `clamp(2rem, 4vw, 3rem)` | 400 | 1.1 | Page hero headlines |

Typography rules:
- Body weight 300 for supporting/descriptive paragraphs; weight 400 for higher-emphasis body copy
- Stat values (numbers) use Bebas Neue at 24px–32px, amber-500 color
- Form submit button: Barlow Condensed, 16px, semibold (`font-semibold`), `letter-spacing: 0.06em`
- No mixing of Bebas Neue and Barlow Condensed on the same line

Source: globals.css `@theme inline`, body styles, SustainabilitySnapshot.tsx, ContactForm.tsx.

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `bg-ink-900` | `#111410` | Primary page/section background |
| Secondary (30%) | `bg-ink-800` | `#1c1e18` | Cards, secondary sections, form inputs (`bg-ink-900/80`), alternate section bg (`bg-ink-800/50`) |
| Accent green (8%) | `green-500` / `green-400` | `#3a9e64` / `#52c17e` | Primary CTA buttons, focus rings, progress bar fill (final milestone), SVG strokes, WhatsApp CTA, checkmark icons |
| Accent amber (2%) | `amber-500` | `#f59e0b` | Section label text, stat numbers, EPR percentage values, progress bar secondary fill, SVG return arc stroke |
| Destructive | `red-400` | — (Tailwind default) | Inline validation error messages only (`text-red-400`) |

Color rules:
- Amber is reserved for: section labels (`.section-label`), numerical stat values, EPR mandate percentages, IOCL/trust callout accent text. Never used for interactive buttons.
- Green is reserved for: primary CTA buttons, progress bars, focus outlines, success states, cert card borders on hover.
- Card borders use `border-ink-600/40` at rest; `border-green-500/30` for highlighted/callout cards.
- Cert "In Progress" badge: `bg-amber-500/10 text-amber-400 border border-amber-500/30` — signals aspirational status without using red/warning.
- Promoter cards: `bg-ink-800 border border-ink-600/40` — same surface as product grade cards in Phase 3.
- Section backgrounds alternate: `bg-ink-950` → `bg-ink-900` → `bg-ink-800` — continue the established alternation pattern.
- Background accent glows: `radial-gradient` with `rgba(58,158,100,0.07–0.15)` — subtle depth, consistent with SustainabilitySnapshot pattern.

Accent reserved for: stat numbers, section labels, EPR percentage values, trust callout mono labels, SVG loop return arc, form field focus ring (green-500/60), primary CTA button.

Source: globals.css token list, SustainabilitySnapshot.tsx, ContactForm.tsx.

## Per-Page Visual Contract

### /about

**Hero:** dark full-width (`bg-ink-950`) with `gradient-mesh` overlay. Headline uses Display typography (Bebas Neue, clamp 2rem–3rem). Lead copy uses Barlow weight 300.

**Two-section layout:**
1. Company & Credibility — Barlow body text, structured paragraphs. Milestone framing callout block: `border-l-2 border-green-500 pl-4` with mono label "PLANT COMMISSIONING" in amber. No visual timeline.
2. Market Opportunity / Why Now — may include 1–2 EPR stat callouts (amber number + Barlow label) using the StatCard pattern.

**Promoter cards:** Horizontal layout (preferred for readability). `bg-ink-800 rounded-xl border border-ink-600/40 p-6`. Name in Barlow Condensed 20px semibold, role in JetBrains Mono 12px amber-500, bio in Barlow 14px ink-200. Avatar area: 56×56px rounded circle placeholder (`bg-ink-700 rounded-full`) — no real photo required for v1.

**"Why now?" stat callout (Claude's discretion — include):** Single row of 2 stats: EPR mandate (50% by FY2031) and India market size. Amber number + Barlow label. Justifies the setup-phase timing for investors.

### /quality

**Hero:** `bg-ink-900 py-28 pt-36 lg:pt-40`. Section label `01 · QUALITY`. Heading frames it as "Certified Standards, Proven Process" or equivalent. No hero image.

**Cert cards (3 total):** Grid `lg:grid-cols-3 gap-6`. Each card: `bg-ink-800 rounded-xl border border-ink-600/40 p-6`. Icon at top (lucide-react: `ShieldCheck` for PCB, `Award` for BIS, `BadgeCheck` for ISO), cert name in Barlow Condensed 18px semibold, issuing body in JetBrains Mono 11px ink-400, description in Barlow 14px ink-200.

**"In Progress" badge:** Positioned top-right of each card. `text-xs font-mono tracking-wider px-2 py-0.5 rounded-full bg-amber-500/10 text-amber-400 border border-amber-500/30`. Text: "PURSUING" (preferred over "In Progress" — more confident).

**Lab testing methodology section:** `bg-ink-800/50` background. Three methodology cards side by side (Viscosity Index, Sulfur Content, Flash Point). Each: icon + test name (Barlow Condensed 16px) + method description (Barlow 14px ink-200). No numerical values.

### /sustainability

**Hero:** `bg-ink-950` with `gradient-mesh` overlay. Environmental opportunity framing ("Turning India's Used Oil Problem Into an Opportunity"). Headline Display typography. Lead copy weight 300.

**EPR Progress Bars (client component `EPRProgressBars.tsx`):**
- Architecture: IntersectionObserver (threshold 0.2) + CSS transition — exact same pattern as `SustainabilitySnapshot.tsx`. Do NOT use Framer Motion here.
- 4 milestones: FY2025 (5%), FY2027 (15%), FY2029 (30%), FY2031 (50%).
- Bar height: `h-3` (12px), track `bg-ink-700 rounded-full`, fill green-700 for intermediate / `bg-gradient-to-r from-green-600 to-green-400` for FY2031 final bar.
- Percentage value: Bebas Neue 24px amber-500. Year label: JetBrains Mono 14px ink-200.
- Section layout: 2-column grid on desktop (narrative text left, bars right) — mirrors SustainabilitySnapshot.

**CO₂ savings block:** Research-based. Executor must find and cite a published figure (search "CO₂ savings re-refined base oil vs virgin lifecycle analysis"). Present as single stat: amber value + Barlow label + source citation in JetBrains Mono 10px ink-400 below.

**Circular Economy SVG (client component `CircularEconomySVG.tsx`):**
- 4 nodes: "Used Oil Collected" → "Re-Refined at Santosh" → "Back Into Engines" → "Collected Again"
- Animation: Framer Motion `animate` + `transition={{ repeat: Infinity }}` for SVG strokeDashoffset loop on connecting arrows. Node boxes appear on scroll via IntersectionObserver + opacity/translateY CSS transition.
- Style: SVG node boxes `fill="#1c1e18" stroke="#2d7a4e" strokeWidth="1.5"`. Arrows `stroke="#3a9e64"`. Return arc `stroke="#f59e0b" strokeDasharray="4 3"`. Text: Barlow Condensed labels inside nodes.
- Viewbox: `0 0 400 400` (circular/square layout for 4-node loop, unlike the 3-node horizontal in SustainabilitySnapshot).
- Reduced motion: Framer Motion respects `prefers-reduced-motion` via its default behavior; globals.css `@media (prefers-reduced-motion)` covers the CSS transition bars.

**IOCL MOU callout:** Reuse exact pattern from SustainabilitySnapshot — `border-l-1 border-green-500 bg-green-900/10 rounded-xl p-5`.

### /collect

**Hero:** `bg-ink-950`. Warm/accessible tone. Headline: "We pick up. You get paid. No hassle." — Bebas Neue Display typography. Subhead: "Turn your used engine oil into income. We handle collection, paperwork, and payment." — Barlow weight 300 ink-200.

**How it works (process steps):** 3–4 steps in horizontal card row on desktop, stacked on mobile. Same card style as AudienceCards (not ProcessSteps sticky scroll — this is a simple static list). Step number in JetBrains Mono amber-500. Step title in Barlow Condensed 18px semibold. Description in Barlow 14px ink-200.

**Service area:** Text list, not a map. `bg-ink-800 rounded-xl border border-ink-600/40 p-6`. Section label "SERVICE AREAS" in JetBrains Mono amber. List of 7 cities in Barlow 15px ink-100, each preceded by a `green-500` dot or `CheckCircle` icon (lucide-react, 16px).

**WhatsApp CTA block (page-specific, per D-09):** Distinct from floating button. Full-width callout `bg-green-900/20 border border-green-500/30 rounded-2xl p-8`. WhatsApp icon (SVG, not lucide). Button: `bg-green-500 hover:bg-green-400 text-white px-8 py-4 rounded-xl` at 20px Barlow Condensed semibold. Text: "Chat on WhatsApp →". Links to `wa.me/919810121438`. Placed immediately after the pickup form success state AND as a standalone section above the footer.

**Pickup scheduling form (client component `PickupForm.tsx`):**
- Fields: Business name (required), Phone (required), Location/address (optional), Quantity available (optional, free text), Preferred frequency (radio/select: One-time pickup / Regular — weekly / Regular — monthly / Not sure yet).
- Validation: Same pattern as ContactForm — `useState errors` object, `trim()` empty check, `noValidate`, `mt-1 text-xs text-red-400` inline messages.
- Input style: `bg-ink-900/80 border border-ink-600/60 rounded-lg px-4 py-3 text-sm text-ink-100 focus:border-green-500/60 focus:outline-none transition-colors` — exact ContactForm classes.
- Success state: Green checkmark circle + "We'll be in touch within 24 hours." + prominent WhatsApp CTA link (`wa.me/919810121438`).
- Submit button: Full-width, `bg-green-500 hover:bg-green-400`, Barlow Condensed semibold, "Schedule Pickup →".
- API target: `POST /api/collect-pickup` (new route, mirrors sample-request/route.ts, writes to "Collect Pickup" sheet tab).

## Interaction Contracts

### Scroll-triggered animations (all pages)

| Component | Trigger | Mechanism | Duration |
|-----------|---------|-----------|----------|
| EPRProgressBars (progress fill) | IntersectionObserver threshold 0.2 | CSS `transition-all duration-1000` with staggered `transitionDelay` per bar | 1000ms per bar, 200ms stagger |
| CircularEconomySVG (node reveal) | IntersectionObserver threshold 0.2 | CSS opacity 0→1, translateY 15px→0 | 500ms, 100ms stagger |
| CircularEconomySVG (arrow loop) | On mount (always animating once visible) | Framer Motion `strokeDashoffset` animate loop | repeat: Infinity, duration: 3s |
| Promoter cards (/about) | IntersectionObserver threshold 0.2 | CSS opacity + translateY — same utility pattern as SustainabilitySnapshot stat cards | 500ms, 100ms stagger |
| Cert cards (/quality) | IntersectionObserver threshold 0.2 | CSS opacity + translateY | 500ms, 150ms stagger |

Reduced motion: globals.css `@media (prefers-reduced-motion: reduce)` sets all animation/transition durations to 0.01ms. Framer Motion respects this natively.

### Form interactions

| State | Visual |
|-------|--------|
| Field at rest | `border-ink-600/60` |
| Field focused | `border-green-500/60` (CSS `focus:border-green-500/60`) |
| Field error | `border-red-400/60` (add on validation failure) + `mt-1 text-xs text-red-400` inline message below field |
| Submit loading | Button disabled, opacity-70, cursor-not-allowed — no spinner required (form is fast) |
| Submit success | Replace form with success state: checkmark icon + heading + body text + WhatsApp link (on /collect) |
| Submit error | Graceful degradation — server write failure shows success to user per Phase 3-05 decision |

### Navigation

Add all 4 new pages to Nav and Footer. Nav links (existing pages): Home, Products, Process, About, Sustainability, Quality, Collect. Footer: same 4 new pages added to site links column. No new CTA variants needed.
