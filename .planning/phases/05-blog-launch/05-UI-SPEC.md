## Spacing Scale

All values are multiples of 4px. Standard 8-point scale in use across all prior phases.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag padding, inline spacing |
| sm | 8px | Compact element spacing, card meta rows |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section sub-spacing, card padding |
| xl | 32px | Column gaps, hero content gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px (py-16 / py-24) | Page-level section padding |

Exceptions:
- Blog post reading body: `max-w-[65ch]` constraint (character-width, not pixel) — per D-10
- Nav touch target: minimum 44px height — accessibility minimum, consistent with prior phases

## Typography

**Blog pages follow D-11 exactly.** Post body uses Barlow 16px; headings inside post body use Bebas Neue.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Barlow | 16px | 400 (regular) | 1.6 | Post body prose, card excerpts, form labels |
| Label | JetBrains Mono | 12px | 400 (regular) | 1.0 | Section labels (.section-label), tag badges, dates, read time |
| Heading | Barlow Condensed | 16px | 700 (bold) | 1.25 | Post card titles on listing page |
| Display | Bebas Neue | clamp(2rem, 4vw, 2.8rem) | 400 (normal — Bebas has no weight variants) | 1.1 | Page hero titles, post hero title, H2/H3 inside post body |

Notes:
- Bebas Neue has no bold variant; weight 400 is its only functional weight
- Post body H2: Bebas Neue, ~28px (text-2xl / text-3xl), line-height 1.1
- Post body H3: Bebas Neue, ~20px (text-xl), line-height 1.1
- Card excerpt text: Barlow 14px, weight 300 (established in existing LatestInsights.tsx)

## Integration UI States

### Resend Email (additive — no new UI)

Resend fires after Google Sheets write succeeds. No new UI state needed — existing form success/error states remain unchanged. If Resend fails, user still sees the green success confirmation from the existing pattern.

### PostHog (no UI)

PostHog is an invisible integration. The `<Script>` tag added to layout.tsx has no rendered output. Event captures (`posthog.capture()`) are called silently from existing form submit handlers.

### Sanity / Blog (new UI)

- **Loading:** Server Components render at request time; no client loading spinners needed. ISR handles stale-while-revalidate invisibly.
- **Not found:** `notFound()` function called when Sanity returns no post for a given slug — triggers Next.js 404 page.
- **Zero posts:** InsightsListing renders the empty state copy when `posts.length === 0`.

## Accessibility Contract

- All post card links: full card is the link target (no nested interactive elements)
- Read time: aria-label="4 minute read" on the time element
- Section labels: aria-hidden="true" (decorative numbering)
- Tag badges: aria-label="{tag} article" or included in the card's accessible name via heading
- Focus visible: 2px solid #3a9e64, offset 2px (established in globals.css)
- Images: alt text required on all Sanity featuredImage renders — use post title as alt if no caption
- Reduced motion: existing `@media (prefers-reduced-motion: reduce)` in globals.css covers all transitions
